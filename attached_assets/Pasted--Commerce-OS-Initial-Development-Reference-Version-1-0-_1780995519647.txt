## 1. Project Overview

### Vision

A multi-tenant SaaS commerce platform enabling merchants to:

- Create and manage online stores
- Manage products, inventory, and orders
- Accept payments (local + global gateways)
- Run POS systems (offline-first, Flutter)
- Manage multiple branches and warehouses
- Run marketing campaigns and loyalty programs
- Manage customers (CRM)
- Track analytics and KPIs
- Connect logistics providers

**Equivalent to:** Shopify + POS + Inventory + CRM + Marketing + Logistics + Analytics

### Business Model

| Revenue Source | Details |
|---|---|
| Subscription plans | Starter, Growth, Business, Enterprise |
| Transaction fees | Percentage per order |
| SMS charges | Usage-based billing |
| Enterprise services | Custom development |
| Setup charges | Store onboarding |
| POS hardware | Scanner, printer, cash drawer |

## 2. Architecture Overview

```
Internet
  ↓
Cloudflare (CDN, DDoS, WAF)
  ↓
Load Balancer (health checks, routing)
  ↓
API Gateway (auth, rate limiting, tenant resolution, routing)
  ↓
Microservices Layer (16 services)
  ↓
Database Layer       Storage Layer        Analytics Layer
(PostgreSQL, Redis)  (MinIO, Elastic)     (Kafka, aggregations)
```

### Technology Stack

| Layer | Technology |
|---|---|
| API Gateway | Kong (or custom NestJS) |
| Microservices | Node.js / NestJS |
| Databases | PostgreSQL (primary), Redis (cache/sessions) |
| Message bus | Apache Kafka |
| Search | Elasticsearch |
| File storage | MinIO → Cloudflare CDN |
| Container orchestration | Kubernetes (AWS EKS) |
| Infrastructure | Terraform, Helm |
| CI/CD | GitHub Actions |
| Observability | Prometheus, Grafana, Loki, Jaeger, OpenTelemetry |

## 3. Tenant & Multi-Tenancy Model

Every merchant is a **Tenant**. All data is scoped by `tenant_id`.

```
Tenant
├── Stores          (online storefronts, custom domains)
├── Warehouses      (stock locations)
├── Branches        (physical locations linked to warehouses)
├── Users           (RBAC, 7 roles)
├── Customers       (CRM, loyalty, segments)
├── Orders          (9-stage lifecycle)
├── Inventory       (6 stock states per variant per warehouse)
├── Themes          (store builder JSON, draft/published)
└── Domains         (custom per store, resolved via Host header)
```

### Isolation Strategy

- **Phase 1–5:** Shared tables with `tenant_id` column + PostgreSQL Row Level Security (RLS)
- **Enterprise tier:** Schema-per-tenant (`search_path` switching)
- **White-label / Phase 6:** Database-per-tenant (dedicated RDS instance)

### RLS Enforcement (Shared Tables)

```sql
-- Enable on every table
ALTER TABLE products ENABLE ROW LEVEL SECURITY;

-- Policy
CREATE POLICY tenant_isolation ON products
  USING (tenant_id = current_setting('app.tenant_id')::uuid);

-- Set at start of every transaction (via common-library middleware)
SET LOCAL app.tenant_id = '<uuid-from-jwt>';
```

**The `common-library` middleware must inject `tenant_id` from JWT on every request. No microservice queries without it.**
